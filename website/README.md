# Documentation site for `@agent-assembly/sdk`

This directory contains the Docusaurus 3 application that builds the public documentation
site at <https://ai-agent-assembly.github.io/node-sdk/>.

Markdown content lives **outside this directory**, at `../docs/`. The Docusaurus app reads
it via `docs.path: '../docs'`. The TypeDoc-generated API reference is regenerated by
`docusaurus-plugin-typedoc` on every build into `../docs/api/` (gitignored).

## Local development

The SDK's `node_modules` must be installed at the repository root (so TypeDoc can resolve
`@types/node` and your local `tsconfig.build.json`):

```bash
# from the repository root
pnpm install
```

Then install and run the docs site:

```bash
cd website
pnpm install
pnpm start          # local dev server with hot reload, http://localhost:3000
```

## Production build

```bash
cd website
pnpm build          # outputs static files into website/build/
pnpm serve          # preview the production build locally
```

## Deployment

The `publish-docs.yml` GitHub Actions workflow builds the site on every push to `master`
and deploys to the `gh-pages` branch. Manual deployment is not necessary; if you need to
trigger a deploy manually, run the workflow via the **Actions** tab using
`workflow_dispatch`.

## Editing content

| Type | Where |
| ---- | ----- |
| Long-form guides (intro, architecture, etc.) | `../docs/*.md` |
| Sidebar order and grouping | `./sidebars.ts` |
| Site title, navbar, footer, theme | `./docusaurus.config.ts` |
| Custom CSS | `./src/css/custom.css` |
| API reference | auto-generated from `../src/index.ts` — do not edit by hand |
